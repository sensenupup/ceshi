二、用户注册入驻流程
flowchart TD
    A[访客进入能力超市首页] --> B[点击"客户入驻"]
    B --> C[填写注册信息]
  <br>
    C --> C1[Step1: 机构信息<br/>机构名称/统一社会信用代码<br/>机构类型/所属行业<br/>所在地区/机构地址]
    C1 --> C2[Step2: 管理员信息<br/>姓名/手机号/邮箱<br/>部门/职位]
    C2 --> C3[Step3: 账号设置<br/>登录密码/确认密码]
    C3 --> C4[Step4: 确认提交<br/>汇总展示所有信息]
  <br>
    C4 --> D{提交入驻申请}
    D --> E[申请进入平台运营方<br/>入驻审核队列]
    E --> F{运营方审核}
    F -->|审核通过| G[账号激活<br/>同时成为能力申请方]
    F -->|驳回| H[返回修改<br/>通知用户重新提交]
    H --> C
  <br>
    G --> I[登录后进入概览面板]
    G --> J[用户中心可查看机构<br/>福建医科大学附属一医院]

四、能力提供方 — 接入与发布流程
flowchart TD
    A[能力提供方登录] --> B[进入"接入申请"]
  <br>
    B --> C[Step1: 基本信息]
    C --> C1[能力名称]
    C --> C2[能力类型<br/>智能体/大模型/医疗服务/数据服务]
    C --> C3[交付形态<br/>私有化部署包/API/元数据/服务]
    C --> C4[封面上传 png/jpg/webp]
    C --> C5[能力介绍视频上传 选填]
    C --> C6[标签填写<br/>输入回车添加]
  <br>
    C --> D{根据交付形态<br/>进入不同配置}
  <br>
    D -->|私有化部署包| E1[上传部署包<br/>上传部署文档<br/>填写环境要求]
    D -->|API| E2[选择API类型<br/>填写接口地址<br/>上传API文档]
    D -->|元数据| E3[上传Schema文档<br/>上传数据样例<br/>填写字段说明]
    D -->|服务| E4[上传服务流程说明<br/>上传SLA文档]
  <br>
    E1 --> F[Step3: 确认提交<br/>汇总全部信息]
    E2 --> F
    E3 --> F
    E4 --> F
  <br>
    F --> G[提交至平台运营方<br/>资质审核队列]
    G --> H{运营方审核}
    H -->|审核通过| I[进入"能力对接"]
    H -->|驳回| C
  <br>
    I --> I1[配置接口/认证/超时]
    I --> I2[连通性测试]
  <br>
    I2 --> J[进入"联调测试"]
    J --> J1[自动生成测试用例]
    J --> J2[单个执行/批量执行]
    J --> J3[查看测试结果]
  <br>
    J3 --> K[进入"上架管理"]
    K --> K1[提交上架申请<br/>选择目标专区<br/>填写变更说明]
    K1 --> L[运营方上架审核]
    L -->|审核通过| M[能力上线至超市首页]
